# 流程与公式规则

规定 3.4 系统流程说明的步骤编号方式，以及公式在文档中的位置决策。

## 系统流程：S1-Sn 步骤式

### 编号体系
采用专利说明书"具体实施方式"常用的 **S 编号**：

| 层级 | 格式 | 示例 |
|------|------|------|
| 主阶段 | **S1**~**S4**（粗体） | **S1 资源加密与策略部署** |
| 子步骤 | **S11**~**S1n**（粗体，列表项） | **S11**：资源提供方登记资源元数据 |
| 子-子步骤 | **S111**~**S11n** | **S121**：执行第一级规则筛选 |

### 写法要求
- 每个子步骤承担**单一明确动作**，不要在一个 S 里塞多件事
- 关键子步骤可点明"为什么这么做"（一句话，不展开论证）
- 用 markdown 列表（`-`）承载子步骤，主阶段用粗体行

### 示例
```markdown
**S1 资源加密与策略部署**

- **S11**：资源提供方登记资源元数据，所述元数据包括资源类型、敏感度等级、合规要求标签。
- **S12**：策略推荐引擎根据所述元数据从策略模板库匹配 CP-ABE 访问策略，包括：
  - **S121**：执行第一级规则筛选……
  - **S122**：对命中的多个候选模板计算相似度……
- **S13**：策略确认后执行混合加密：
  - **S131**：随机生成 AES 密钥 K 加密数据本体；
  - **S132**：用 CP-ABE 按策略封装 K……
```

## 公式位置：集中定义，编号引用

### 决策结论：公式集中放符号与公式节（如 3.4.1），不在流程步骤内重复本体

**依据**：交底书中的公式具有"一处定义、多处引用"的网状引用结构——每个公式被 3.3 模块说明、3.4 流程、第五章保护点、第六章实施例等多处引用。集中定义使公式有**唯一固定位置**，所有引用处指向同一处。

### 引用方式
流程步骤内只写**编号引用**，不抄公式本体：

```
✓ **S25**：系统生成属性绑定挑战 nonce（式 (1)），将当前属性集 Attr 绑定其中下发给设备。
✗ **S25**：系统生成属性绑定挑战 nonce = H(r ‖ H(Attr))，……（不要在步骤内写公式本体）
```

### 公式节的布局
3.4.1（或同等小节）集中放置：
1. **符号表**（分组：系统参数/密钥/消解/版本链等，每个符号给含义+下标/量纲）
2. **核心公式**（编号式 (1)~(n)，每个公式前有"**式 (n)**：用途说明"小标题）

## 块级公式渲染要求（与 md_to_docx 配合）

md_to_docx 的块级公式解析器只识别**独占一行的 `\[`**（`line.strip()=="\\["`）。因此块级公式必须写成：

```markdown
\[
公式体
\]
<!-- ![公式](math_figures/eq_00X.png) -->
```

**不要**写成单行 `\[\dots\]`——会被当作普通文本，公式以 LaTeX 残留在 docx 中。

## 嵌套括号行内公式（渲染陷阱）

含**内层未转义括号**的行内公式（如 `\(nonce = H(r \| H(Attr))\)`）会导致：
- math_render 的正则 `\\\(((?:\\.|[^)])+?)\\\)` 在内层 `)` 处提前截断，无法正确匹配
- 该公式无法生成 PNG，以 LaTeX 文本残留

**处理**：将此类公式改为**编号引用**（"式 (1)"，公式本体在 3.4.1 块级给出），或改为文本表述。
