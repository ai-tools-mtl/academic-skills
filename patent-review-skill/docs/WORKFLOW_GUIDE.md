# 工作流使用指南

## 概述

Patent Review Skill 采用四阶段穿透式审查工作流，通过 AI 辅助与人工介入的结合，实现专利客体适格性的全面审查。

## 各阶段详解

### 阶段一：初步穿透筛查

**目的**：快速识别高风险申请，为后续深度审查提供方向。

**节点列表**：

| 节点 | 类型 | 功能 |
|------|------|------|
| 1.1 特征提取与标注 | AI | 提取权利要求1全部技术特征 |
| 1.2 名实分离风险识别 | AI | 识别名实分离风险点 |
| 1.3 数据含义初判 | AI | 初步判断数据层次（L1-L5） |
| 1.4 问题性质初判 | AI | 判断问题属于技术/算法/商业 |
| 1.5 风险报告生成 | AI | 生成结构化风险报告 |
| 1.6 审查员复核 | 人工 | 复核报告，确认风险点 |

**退出条件**：
- `risk_level == '低'` → 结案
- `risk_level in ['中', '高']` → 阶段二

### 阶段二：深度穿透审查

**目的**：深入剖析三要素的技术属性。

**节点列表**：

| 节点 | 类型 | 功能 |
|------|------|------|
| 2.1 技术问题穿透 | AI | 执行删除测试 |
| 2.2 技术手段穿透 | AI | 检索特定技术关联 |
| 2.3 技术效果穿透 | AI | 匹配效果指标 |
| 2.4 闭环断层检测 | AI | 检测三要素断层 |
| 2.5 三要素报告生成 | AI | 生成分析报告 |
| 2.6 疑点确认 | 人工 | 确认技术属性 |

**退出条件**：
- 任一要素非技术 → 终止（不授权）
- 三要素均技术 → 阶段三

### 阶段三：锚定穿透验证

**目的**：量化验证与技术领域的锚定程度。

**节点列表**：

| 节点 | 类型 | 功能 |
|------|------|------|
| 3.1 数据含义量化 | AI | 精确归入L1-L5 |
| 3.2 关联度量化 | AI | 归入G0-G4 |
| 3.3 综合矩阵判断 | AI | 执行L×G判断 |
| 3.4 验证报告生成 | AI | 生成验证报告 |
| 3.5 存疑分析 | 人工 | 分析存疑情形 |

**L×G 判断矩阵**：

| 组合 | 结果 | 说明 |
|------|------|------|
| L4 且 G3+ | 锚定成功 | 技术属性明确 |
| L3 且 G2- | 锚定失败 | 非技术属性 |
| 其他 | 存疑 | 需人工分析 |

### 阶段四：案例穿透与结论生成

**目的**：类案比对，形成最终审查意见。

**节点列表**：

| 节点 | 类型 | 功能 |
|------|------|------|
| 4.1 类案匹配 | AI | 匹配相似案例 |
| 4.2 比对分析 | AI | 五维度比对 |
| 4.3 意见生成 | AI | 生成审查意见初稿 |
| 4.4 最终审核 | 人工 | 最终审核确认 |

## 使用方式

### 完整工作流执行

```python
from workflow.patent_workflow import PatentReviewWorkflow

workflow = PatentReviewWorkflow()

result = workflow.execute(
    patent_application="./case_2024_001.pdf",
    stage="all"
)
```

### 分阶段执行

```python
# 只执行阶段一
result = workflow.execute(
    patent_application="./case.pdf",
    stage="stage1"
)

# 执行到阶段二（包含阶段一）
result = workflow.execute(
    patent_application="./case.pdf",
    stage="stage2"
)
```

### 单节点执行

```python
from workflow.nodes import FeatureExtractionNode

node = FeatureExtractionNode()
result = node.execute(
    patent_application="./case.pdf"
)
```

## 自定义配置

### 修改节点 Prompt

编辑 `workflow/node-prompts.md`，找到对应节点修改 Prompt。

### 添加新节点

1. 在 `patent-review-workflow.json` 中添加节点定义
2. 在 `workflow/nodes/` 中创建节点类
3. 更新 `node-prompts.md`

### 修改关键词库

编辑 `workflow/keyword-libraries.json`。

## 最佳实践

1. **阶段一必须认真**：仔细复核 AI 生成的风险报告
2. **善用退出条件**：低风险案件及时结案
3. **充分利用案例库**：参考相似案例做出判断
4. **保持文档更新**：记录审查决策依据
