# 专利审查技能自我进化机制

## 概述

本技能内置自我进化机制，通过用户反馈和定期追踪最新审查案例，持续完善专利客体适格性审查能力。

## 进化原则

### 第一性原理

所有进化必须从专利法核心定义出发：

```
专利法第二条第二款：
技术方案是对要解决的技术问题所采取的
技术手段构成了技术特征，并且
能够产生相应的技术效果
```

**三要素标准**：
- 技术问题：利用自然规律解决技术问题
- 技术手段：与技术特征功能上相互支持
- 技术效果：获得符合自然规律的技术效果

### 严格流程

```
反馈/案例 → 分类识别 → 第一性原则验证 → 知识更新 → 版本发布
     ↓           ↓              ↓             ↓
  收集器    质量判定     一致性检查      验证测试
```

## 进化来源

### 用户反馈

| 类型 | 说明 | 优先级 |
|------|------|--------|
| 规则修正 | 审查标准错误 | P0 |
| 案例补充 | 缺失重要案例 | P1 |
| 边界澄清 | 边界案例说明 | P2 |
| 错误纠正 | 事实性错误 | P0 |
| 体验改进 | 流程优化建议 | P3 |

### 案例追踪

**来源**：
- 国家知识产权局审查案例库
- 最高人民法院知识产权法庭判例
- 专利复审委员会无效决定
- 审查指南官方示例

**频率**：每周自动检查更新

## 进化流程

### 1. 反馈收集

```python
feedback_collector.submit(
    feedback_type="案例补充",
    content="...",
    evidence=[case_data],
    source="审查员A"
)
```

### 2. 第一性验证

每次更新必须验证：

1. **一致性检验**：新内容是否与专利法第二条一致
2. **完整性检验**：是否覆盖三要素所有情况
3. **可执行性检验**：是否能指导实际审查工作

### 3. 知识更新

```
更新类型：
├── 新增案例 → case-database.md
├── 修正规则 → patent-law-basis.md
├── 更新关键词 → keyword-libraries.json
└── 调整Prompt → node-prompts.md
```

### 4. 版本管理

```
版本格式：v{MAJOR}.{MINOR}.{PATCH}
- MAJOR: 核心法律依据重大变更
- MINOR: 新增案例/规则完善
- PATCH: 修正错误/优化表述
```

## 自我验证机制

### 单元测试

每次更新后自动运行：

```bash
python -m pytest validation/test_knowledge_update.py
```

### 回归测试

使用预设测试用例验证：

1. 授权案例应通过审查
2. 不授权案例应正确识别
3. 边界案例应准确分类

### 一致性检查

```
检查项：
□ 三要素标准一致
□ L1-L5/G0-G4模型一致
□ 节点Prompt与规则一致
□ 案例与规则匹配
```

## 执行命令

```bash
# 提交用户反馈
python evolution/feedback_collector.py submit --type "案例补充" --file feedback.json

# 触发进化流程
python evolution/evolution_engine.py run

# 检查最新案例
python evolution/case_tracker.py update

# 发布新版本
python evolution/version_manager.py release --version "v1.2.0"

# 运行完整验证
python -m pytest validation/ -v
```

## 安全机制

- 所有更新需通过第一性原理验证
- 核心法律依据变更需人工确认
- 保留完整变更历史
- 支持回滚到任意版本
